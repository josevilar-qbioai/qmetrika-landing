Agent Board: gobernanza fail-closed para agentes de IA

Un tablero Kanban donde varios agentes de IA colaboran bajo supervisión humana: broker de aprobación fail-closed, auditoría con cadena de hash, RBAC y adaptador multi-LLM. Nació como banco de pruebas de los pipelines del laboratorio.

colateral agent-board gobernanza
Nota: Agent Board es una línea colateral de QMetrika Labs, no el core. El core es la biocomputación (motor ΔG, EnergyFingerprint). Agent Board nació como la infraestructura que gobierna esos pipelines científicos, y aquí lo contamos como pieza de exploración reutilizable.

El problema

Cuando el coste de generar un texto, un análisis o una decisión con IA cae a casi cero, aparecen cuatro problemas que ninguna API de LLM resuelve por sí sola: gasto ingobernado (los agentes queman tokens sin límite ni visibilidad de coste por tarea), sin rastro de auditoría (no queda registro verificable de quién aprobó qué, cuándo y sobre qué), dependencia de proveedor, y fuga de datos (los datos sensibles salen hacia una API cloud).

En un dominio regulado —datos genómicos de paciente, con requisitos de trazabilidad y soberanía— esos cuatro problemas dejan de ser molestias y pasan a ser bloqueantes.

La idea: el humano decide, el sistema propone

Agent Board es un Kanban de cinco columnas (Queued → Working → Needs Input → Review → Done) donde cada agente hace su parte y el pipeline se detiene en un gate hasta que un humano autoriza. No es decoración: el broker es fail-closed — si nadie aprueba, no se avanza.

Un token gastado y olvidado es gasto. Un token gastado, gobernado y capturado, es infraestructura.

Las piezas

  • Broker de aprobación fail-closed: el flujo se para en cada gate; la decisión humana queda ligada criptográficamente al payload.
  • Auditoría hash-chain tamper-evident: quién, cuándo, qué payload, qué hash. Verificable con un comando — la base de un dossier regulatorio.
  • RBAC declarativo (policy.json) + MCP Gate con decorador @gated.
  • Adaptador universal multi-LLM (~60 líneas): cada agente usa el modelo óptimo —frontera, especializado o local— sin vendor lock-in.
  • Panel de coste dual (multi-LLM vs todo-Opus): hace visible el ahorro por tarea y por modelo.

Por qué nació aquí

No se diseñó en abstracto. Agent Board es la capa que orquesta los pipelines de clasificación de variantes de QMetrika: un Auditor Genómico (modelo barato) descarga y filtra, un Motor Biofísico local computa perfiles ΔG sin que los datos salgan del servidor, un Predictor puntúa, y un gate detiene todo hasta que el genetista revisa. Ese es el caso de uso más exigente que existe —datos de paciente, compliance, human-in-the-loop— y es el que validó el enfoque.

Dónde encaja

Como pieza de producto, Agent Board es la capa de gobernanza para equipos que despliegan agentes sobre datos sensibles o en dominios regulados (salud, legal, finanzas). Hoy tiene el producto construido —broker, MCP Gate, RBAC, auditoría, 12 ADRs, un threat model con 15 amenazas— y un caso de uso interno sólido, pero aún no clientes externos: es una línea de exploración, no una validada en mercado.
